Splash Page

Technical Description

The splash screen serves as the entry point to our climate action website, designed to create an engaging first impression while the main content loads. I implemented this page using several key HTML and CSS techniques:

HTML Structure and Components:

  • Container Structure: I used nested div elements to create a hierarchical layout, with the main splash-container housing all content and providing full viewport coverage.
  • Background Image Implementation: Instead of using CSS background properties, I implemented the background as an img element with the splash-bg class and positioned it absolutely to allow for better accessibility and filter application.
  • Content Organization: The content-wrapper div contains logically grouped sections (logo, welcome text, team information, and loading indicator) to maintain semantic structure.
  • Team Member List: Used an unordered list (ul) with list items (li) to properly structure team member information, enhancing both semantics and screen reader compatibility.

CSS Techniques and Styling:

  • Responsive Design: Implemented multiple media queries to ensure optimal display across different device sizes (mobile, tablet, and laptop), with flexible content sizing and layout adjustments.
  • Visual Enhancements: Applied subtle animations including fadeIn, slideUp, and spin keyframe animations to create a dynamic loading experience without overwhelming the user.
  • Accessibility-Focused Styling: Used high contrast colors defined via CSS variables, appropriate text sizing, and visual separation between content sections to improve readability.
  • Glassmorphism Effect: Created a semi-transparent overlay with backdrop filter for the content wrapper, providing visual depth while maintaining readability of content.

JavaScript Functionality:

  • Timed Redirection: Implemented a setTimeout function that redirects users to the home page after 4 seconds, giving them time to view the splash screen.
  • Dynamic Loading Indicator: Created an animated loading text using setInterval that cycles through displaying 1-3 dots after "Loading your experience" to provide visual feedback during the waiting period.

Accessibility

I prioritized accessibility throughout the splash screen implementation:

  • Alternative Text: All images include descriptive alt attributes, including "Splash Background" for the background image and "Logo" for the SDG logo.
  • Semantic HTML: Used appropriate heading levels (h1) for the main title "Climate Action" and structured content logically for screen readers.
  • Responsive Design: Ensured the page is fully responsive across devices with appropriate text sizing and layout adjustments.
  • Visual Indicators: The loading spinner and dynamic text provide both visual and textual cues about the loading process.
  • Color Contrast: Maintained high contrast between text and background colors using CSS variables to ensure readability for visually impaired users.
  • Dark Mode Support: Implemented automatic adjustments for users who prefer dark mode, reducing eye strain in low-light conditions.

Volunteer Page

Technical Description

The volunteer page serves as a crucial engagement point for our climate action website, designed to inform and motivate potential volunteers while showcasing available programs. I implemented this page using several key HTML and CSS techniques:

HTML Structure and Components:

  • Semantic Markup: I utilized semantic HTML5 elements including <main>, <article>, <section>, and <form> to create a logically structured document that enhances accessibility and SEO.
  • Card-Based Layout: Implemented multiple card components using <article> elements within container <div>s to present program options and reviews in a consistent, visually appealing format.
  • Form Implementation: Created an interactive feedback form with <fieldset>, <label>, and various input types to collect user reviews, including a custom rating scale using radio buttons.
  • Icon Integration: Incorporated Font Awesome icons using the <i> element with appropriate classes to enhance visual communication and provide meaningful visual cues.
  • Dynamic Dropdown: Built a custom sorting dropdown menu with toggle functionality that reveals options upon hover, improving user experience without requiring JavaScript for basic functionality.

CSS Techniques and Styling:

  • CSS Grid Layout: Implemented responsive grid layouts using display: grid with grid-template-columns and minmax() functions for program cards, review cards, and benefits sections that automatically adjust based on screen size.
  • Flex-Based Components: Used display: flex for alignment and spacing in numerous components including the navigation, button containers, and rating scale to ensure proper alignment and spacing.
  • Interactive Elements: Applied hover effects using transform: translateY() and box-shadow changes to create subtle animations when users interact with cards, enhancing engagement without being distracting.
  • CSS Variables: Leveraged CSS variables (custom properties) for colors and common values to maintain a consistent theme throughout the page while facilitating easy updates and theme switching.
  • Comprehensive Media Queries: Implemented detailed breakpoints at 1200px, 992px, 768px, and 480px to ensure optimal viewing across devices, adjusting grid layouts, font sizes, and component arrangements accordingly.
  • Custom Form Styling: Created visually appealing form elements with custom-styled radio buttons using the CSS technique of hiding the default inputs and styling their associated labels.

Special Features:

  • Card Hover Effects: Implemented subtle elevation effects using transform and enhanced box-shadow properties when users hover over program or review cards, creating a tactile, interactive feel.
  • Custom Rating Interface: Designed an intuitive rating system with emoticon icons that scale and change color when selected, providing visual feedback and improving user engagement.
  • Text Truncation: Used -webkit-line-clamp and related properties to elegantly truncate text in card excerpts, maintaining consistent card heights while indicating additional content with ellipses.
  • Interactive Buttons: Created button styles with color inversion on hover, maintaining the color scheme while providing clear visual feedback for interactive elements.

Accessibility

I implemented several accessibility features throughout the volunteer page:

  • Semantic HTML Structure: Used proper heading hierarchy (h1-h3) and semantic elements to ensure screen readers can navigate the content logically.
  • Form Accessibility: Implemented properly associated <label> elements with form controls, ensuring screen reader users understand the purpose of each input.
  • Alt Text for Images: Added descriptive alt attributes for program images, country flags, and decorative elements to provide context for visually impaired users.
  • Color Contrast: Used CSS variables to maintain sufficient contrast between text and background colors, ensuring readability for users with visual impairments.
  • Responsive Design: Created a fully responsive layout that adapts to different screen sizes and orientations, making the content accessible across various devices.
  • Focus Indicators: Maintained visible focus states for interactive elements to assist keyboard navigation users.
  • Icon Support: Complemented icons with text labels to ensure their meaning is clear to all users, regardless of visual ability.

Content Page

Technical Description

The sustainable energy content page serves as an informational resource about renewable energy within our climate action website. I designed this page to effectively communicate key concepts while maintaining visual engagement. Here's how I implemented it using HTML and CSS techniques:

HTML Structure and Components:

  • Semantic Structure: I used the <main> element with a unique ID (id="main-content") to clearly define the primary content area, improving accessibility and SEO by properly identifying the page's core content.
  • Content Sectioning: Divided the page into logical sections using <div> elements with descriptive class names (content, image-container, featured-section, card-container) to organize content thematically.
  • Heading Hierarchy: Implemented a proper heading structure with <h1> for the main title and <h2> and <h3> for subsections, creating a logical document outline for both visual hierarchy and screen readers.
  • Icon Integration: Incorporated Font Awesome icons using the <i> element with appropriate classes (e.g., fas fa-leaf, fas fa-bolt) to enhance visual communication and provide thematic indicators for different content sections.
  • Image Implementation: Placed images within a container using the <img> element with descriptive alt attributes, ensuring accessibility while providing visual content to support the text information.

CSS Techniques and Styling:

  • Flexible Layout: Used display: flex for the image container to create a side-by-side arrangement of images with a vertical divider, enabling efficient use of horizontal space while maintaining visual balance.
  • Image Optimization: Applied object-fit: cover to images to ensure they maintain their aspect ratio while filling their allocated space, preventing distortion regardless of image dimensions.
  • Responsive Constraints: Set max-width: 40% for images to ensure they don't dominate the layout on larger screens while maintaining readability of accompanying text content.
  • Visual Separation: Created a vertical-line element between images using a thin background-color with specific height and width properties to provide clear visual separation without taking excessive space.
  • Content Spacing: Applied margin and padding properties throughout the page to create breathing room between elements, improving readability and visual flow.
  • Border Radius: Used border-radius: 8px on images to soften their appearance and maintain consistency with the rounded design language used throughout the site.

Content Presentation:

  • Card-Based Information: Implemented card layouts in the featured-cards and card-container sections to group related content visually, making information more digestible and scannable for users.
  • Visual Icons: Strategically placed icons within headings and cards to provide immediate visual cues about content topics, enhancing understanding before users read the detailed text.
  • Consistent Spacing: Used <br> elements and margin properties to create consistent spacing between text paragraphs and major sections, improving readability and content flow.
  • Header/Footer Integration: Implemented container divs (id="header-container" and id="footer-container") that are populated via JavaScript includes, ensuring consistent navigation and footer elements across all site pages.
  • Theme Support: Added a theme switcher container that interfaces with the site's theming system, allowing users to toggle between light and dark modes for better readability according to their preference.

Accessibility

I prioritized accessibility throughout the sustainable energy page implementation:

  • Alternative Text: Provided descriptive alt attributes for all images, including "Sustainable Energy" and "Sustainable Energy Infographic" to ensure screen reader users understand the image content.
  • Semantic Structure: Used proper heading hierarchy and semantic HTML elements to create a logical document structure that can be easily navigated with assistive technologies.
  • Complementary Icons and Text: Paired icons with descriptive text rather than relying solely on visual elements, ensuring all users can understand the content regardless of visual ability.
  • Color Contrast: Maintained appropriate contrast between text and background colors using the site's CSS variables system, ensuring readability for users with visual impairments.
  • Responsive Design: Created a layout that adapts to different screen sizes, maintaining readability and functionality across devices, benefiting users with different viewing needs.
  • Clear Content Structure: Organized content in logical sections with appropriate headings and concise paragraphs, making the page easier to understand for all users, including those with cognitive disabilities.
  • Top Padding: Applied padding-top: 50px to the body to ensure content doesn't begin immediately under the fixed header, preventing content from being hidden or difficult to access.

Challenges and Lessons Learned

Implementation Challenges

During the development of these pages, I encountered several technical challenges that required creative problem-solving:

1. Responsive Layout Management

Creating truly responsive layouts that worked seamlessly across all device sizes proved challenging, particularly for the volunteer page with its complex card-based grid system. Initially, the cards would overlap or break at certain viewport widths, creating visual inconsistencies.

Solution: I implemented a comprehensive media query strategy with carefully selected breakpoints (1200px, 992px, 768px, and 480px) and used CSS Grid's minmax() function to create flexible card dimensions. This allowed elements to reflow naturally without fixed widths breaking the layout.

2. Cross-Browser Compatibility

Ensuring consistent rendering across browsers was difficult, especially for modern CSS features like -webkit-line-clamp used for text truncation in card descriptions and custom dropdown styling in the volunteer page.

Solution: I implemented progressive enhancement approaches, providing fallback styling for browsers that didn't support certain features. For text truncation, I added both standard and webkit-specific properties along with appropriate overflow handling.

3. Animation Performance

The splash screen's loading animations initially caused performance issues on lower-end mobile devices, creating a sluggish user experience during the crucial first impression.

Solution: I optimized animations by using CSS transform properties instead of margin/position changes to leverage GPU acceleration. Additionally, I simplified some animations and ensured they were triggered appropriately to minimize resource usage.

Lessons Learned

  • Mobile-First Design: Starting with mobile layouts first proved more effective than retrofitting desktop designs for smaller screens. This approach forced me to prioritize content and functionality early in the design process.
  • CSS Variables Efficiency: Using CSS variables (custom properties) significantly streamlined theme management and made global styling changes much more maintainable. This became particularly valuable when implementing dark mode support.
  • Semantic HTML Importance: Consistently using semantic HTML elements not only improved accessibility but also made styling more logical and debugging easier. The structure provided natural organization that carried through to the final presentation.
  • Performance Testing: Regular testing on different devices revealed performance issues early, allowing for timely optimizations. This was especially important for animation-heavy elements like the splash screen.

Compliance

All pages I created adhere to Janet regulations governing web page publication through careful implementation of the following compliance measures:

Accessibility Compliance

I ensured compliance with WCAG 2.1 Level AA standards, which Janet regulations incorporate as a baseline requirement:

  • Alternative Text: All images across the splash, volunteer, and content pages include descriptive alt attributes to ensure screen reader compatibility.
  • Semantic HTML: Properly structured HTML with appropriate heading hierarchies (h1-h3) creates a logical document outline that assistive technologies can navigate effectively.
  • Color Contrast: Text elements maintain sufficient contrast ratios with their backgrounds using consistent CSS variables, meeting WCAG AA requirements for readability.
  • Keyboard Navigation: All interactive elements are accessible via keyboard, with visible focus states maintained for improved navigation.

Data Protection Compliance

In accordance with Janet's data protection requirements:

  • Minimal Data Collection: The feedback form on the volunteer page collects only necessary information without requesting personally identifiable data beyond what's needed for the specific purpose.
  • Transparent Purpose: Form submissions clearly indicate their purpose through descriptive labels and submission button text.

Security Compliance

I implemented several security measures aligned with Janet's web security guidelines:

  • Resource Integrity: External resources are properly referenced and would be implemented with integrity attributes in a production environment to prevent malicious code injection.
  • Input Validation: Form fields include appropriate attributes to validate input types, helping prevent malformed data submission (though full server-side validation would be implemented in production).

Content and Copyright Compliance

  • Proper Attribution: All external resources including images and icons are appropriately attributed in the References section.
  • Original Content: Written content for the sustainable energy and volunteer pages was created specifically for this project to avoid copyright concerns.

These compliance measures were integrated throughout the development process rather than added as an afterthought, ensuring that all pages meet Janet's regulatory requirements while still delivering an engaging user experience.
