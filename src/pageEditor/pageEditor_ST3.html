Feedback Page


Technical Description

The feedback page is implemented using HTML, CSS, and JavaScript, ensuring a responsive, structured, and user-friendly form with real-time previews.


  • The page uses fieldset, label, and input to group related form fields, enhancing both structure and accessibility.
  • Includes text fields input, a textarea, and a select dropdown to collect user data effectively.
  • Uses paragraph elements with unique IDs to dynamically update the entered data in real time using JavaScript.
  • The .container class ensures a centered layout, and media queries optimize the form for smaller screens.
  • Custom borders, padding, and opacity effects enhance the UI. Inputs dynamically change styles based on validity (:valid and :invalid).
  • Inputs and buttons have hover effects (opacity, scale, transition) for an interactive feel.
  • A visually appealing rating system with hidden radio inputs and styled labels improves user experience.


Accessibility Features

  • Label elements explicitly define input fields, improving readability for screen readers.
  • Background colors and text colors provide sufficient contrast for readability.
  • Uses .error-msg to provide immediate feedback for invalid inputs, enhancing accessibility for users with cognitive impairments.

Team Page

Technical Description

The Team page is implemented using HTML,CSS, ensuring a responsive, structured, and user-friendly Team Page with all of the necessary details of team members.


  • The page uses main, div, h1, p, and img tags to create a well-structured and readable layout.
  • Each member's details are wrapped in a div class="team-member", with an img for the profile picture and a div class="member-details" for names, roles, contributions, and social media links.
  • Font Awesome icons provide links to GitHub, LinkedIn, and a personal editor page for each member.
  • Image container uses display:flex to arrange team members in a row.
  • Hover selector changes opacity, scales the image (as a zoom-in effect) and displays the member details

Accessibility Features

  • Images have descriptive alt attributes for visually impaired users.
  • Media query adjusts layout for mobile devices
  • Social media links with icons.

Content Page - ST3

Technical Description

The Content page is implemented using HTML and CSS, ensuring a responsive, structured, and user-friendly content page with all of the necessary details.


  • The img tag is used to display the "SDGs" image, with border-radius and opacity for styling
  • h1, h2, and h3 tags are used for titles and subtitles, styled with color to align with the theme.
  • Content is divided using div , encapsulating various sections like "Climate Action" and "Climate Change".
  • Flexbox used for horizontal alignment of content like cards and featured sections.
  • Body and head colors are set using CSS variables.
  • Hover effects are used for added interactivity.

Accessibility Features

  • The "Back to Top" button is clearly visible and accessible for keyboard navigation, ensuring that users with motor disabilities can easily interact with the page.
  • Use of high contrast colors for text and background to improve readability for users with visual impairments.
  • Descriptive alt text for images to assist screen readers in understanding image content.
