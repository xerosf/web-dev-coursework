Technical Description

Responsive Components

One of my major contributions was developing responsive components that ensure our website functions flawlessly across all device sizes. These components include the header, footer, and theme switcher. Below I'll explain the techniques used for each component:

Responsive Header

The header was implemented with a focus on responsive design using CSS Grid and Flexbox. For the mobile view, I used a hamburger menu that dynamically transforms into an "X" when clicked, providing a modern and intuitive navigation experience.

CSS Grid Flexbox CSS Transitions Media Queries JavaScript Events

Key HTML and CSS techniques used:

  • Position: fixed - Ensures the header stays at the top of the viewport while scrolling
  • Backdrop Filter - Creates a frosted glass effect for modern UI aesthetics
  • CSS Grid Template Columns - Restructures the layout at mobile breakpoints
  • Transform & Transition - Used for smooth animation of the hamburger menu
  • JavaScript Toggle - Controls the display of dropdown menus and mobile navigation

For larger screens, the header displays a horizontal navigation menu. At the 768px breakpoint, it transforms into a mobile-friendly layout with a hamburger menu that triggers a dropdown navigation panel:

@media screen and (max-width: 768px) { header { display: grid; grid-template-columns: 1fr 1fr 1fr; align-items: center; } .hamburger-menu { display: block; } nav { position: absolute; top: 100%; left: 0; width: 100%; max-height: 0; overflow: hidden; transition: max-height 0.3s ease; } }

Responsive Footer

The footer was designed with a multi-column grid layout that adapts to different screen sizes. On mobile, it transforms into a stacked layout with columned content for better readability and interaction.

CSS Grid Auto-fit Minmax Function Media Queries

Key HTML and CSS techniques used:

  • Grid Template Columns - Uses repeat(auto-fit, minmax(150px, 1fr)) for responsive column behavior
  • Flexbox - Aligns social icons and buttons within the footer
  • Media Queries - Changes grid layout from multi-column to two columns at 768px breakpoint
  • Text Alignment - Shifts from left-aligned to center-aligned on mobile for better readability

The following CSS creates a responsive grid that automatically adjusts the number of columns based on available space:

.footer-links { display: grid; grid-template-columns: repeat(auto-fit, minmax(150px, 1fr)); gap: 2rem; margin-top: 2rem; } @media screen and (max-width: 768px) { .footer-links { grid-template-columns: repeat(2, 1fr); text-align: center; } .footer-column.social { flex-direction: column; align-items: center; } }

Theme Switcher

I implemented a dynamic theme switcher that allows users to toggle between light and dark modes. This component uses JavaScript to save the user's preference in localStorage and CSS custom properties (variables) for theming.

CSS Variables localStorage API Event Listeners CSS Transitions

Key techniques used:

  • CSS Custom Properties - Defined theme colors as variables that can be swapped
  • HTML data-theme Attribute - Used to control which theme is currently active
  • localStorage - Persists user's theme preference across page visits
  • Event Listeners - Detects changes to the theme toggle switch
  • Transition Prevention - Added logic to prevent transition flicker on page load

The theme switcher JavaScript handles theme persistence and smooth transitions:

function switchTheme(e) { if (e.target.checked) { document.documentElement.setAttribute('data-theme', 'dark'); localStorage.setItem('theme', 'dark'); } else { document.documentElement.setAttribute('data-theme', 'light'); localStorage.setItem('theme', 'light'); } }

Challenges and Lessons Learned

Responsive Navigation Challenges

One of the significant challenges I faced was implementing a responsive navigation system that worked seamlessly across all device sizes. The dropdown menus were particularly challenging, as they needed to behave differently on desktop (hover) versus mobile (click).

Solution: I resolved this by using JavaScript to detect screen size and implementing different event handlers based on the current viewport. For mobile views, I used click events with toggle functionality, while desktop used hover events. This taught me the importance of considering different interaction models across devices.

Theme Switching Implementation

Implementing the theme switching functionality presented challenges, particularly with preventing the "flash of unstyled content" when a user with a saved dark theme preference loads the page.

Solution: I solved this by applying the theme immediately during page loading using inline JavaScript that checks localStorage before the DOM fully loads. This prevents the initial flash of the default theme. I learned the importance of optimizing the critical rendering path and how to properly use localStorage for persistent preferences.

Cross-browser Compatibility

Ensuring consistent appearance and behavior across different browsers proved challenging, especially with newer CSS features like backdrop-filter.

Solution: I implemented feature detection and fallbacks for browsers that don't support certain CSS properties. This experience taught me the value of progressive enhancement and not relying too heavily on cutting-edge features without proper fallbacks.

Component Loading and Integration

Designing a system for loading shared components (header, footer, theme switcher) across multiple pages while ensuring proper initialization of JavaScript functionality was difficult.

Solution: I developed a component loading system using the Fetch API that loads HTML components and then initializes their associated JavaScript. This taught me about asynchronous loading patterns and the importance of proper event handling to ensure components are fully loaded before scripts run.

Accessibility

Ensuring our website is accessible to all users was a priority throughout development. I implemented several accessibility features across the pages I created:

  • Semantic HTML Structure: Used appropriate elements like <header>, <footer>, <nav>, and proper heading hierarchy to improve screen reader navigation.
  • Alternative Text: Added descriptive alt text for all images to provide context for screen reader users.
  • Responsive Design: Ensured content is accessible across all device sizes, with text that scales properly without breaking layouts.
  • Focus Management: Implemented proper focus management for interactive components like the mobile navigation menu.
  • ARIA Attributes: Used ARIA roles, states, and properties where needed to enhance accessibility of complex components.

For the Targets page specifically, I ensured that the card-based layout remains accessible by maintaining logical tab order and providing sufficient color contrast for the icon elements. The responsive design ensures that content remains accessible regardless of screen size.
